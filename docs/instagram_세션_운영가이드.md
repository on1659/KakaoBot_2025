# Instagram 세션 운영 가이드

- 대상: 봇 운영자
- 관련: [instagram_기능_복구_작업계획.md](instagram_기능_복구_작업계획.md) §4.3

> **먼저 알아둘 것: 대부분의 경우 이 문서가 필요 없다.**
> 봇은 기본적으로 **로그인 없이** 공개 게시물·릴스의 대표 이미지를 가져온다
> (embed 페이지 스크래핑). 세션 파일은 **비공개 게시물처럼 embed로 안 되는 경우에만**
> 쓰이는 선택적 폴백이다. 공개 게시물만 다룬다면 아래 절차는 건너뛰어도 된다.

Instagram 이미지 기능은 비밀번호 로그인을 쓰지 않는다. 폴백이 필요할 때는 브라우저에서
만든 로그인 세션을 파일(`saved/instagram/session-<계정명>`)로 저장해 두고 봇이 재사용한다.

## 세션 파일 만들기 / 갱신하기

1. **봇(main.py)을 완전히 종료한다.**
   봇이 실행 중이면 카카오톡 창 강제 전면화 + 전역 키 입력 때문에, 아래에서
   입력하는 값이 카카오톡 방으로 전송될 수 있다. 반드시 먼저 종료한다.
2. 아무 브라우저에서 instagram.com에 봇 계정으로 로그인한다.
3. 개발자도구(F12) → Application(Chrome) 또는 저장소(Firefox) → Cookies
   → `https://www.instagram.com` 에서 두 값을 복사한다:
   - `sessionid`
   - `csrftoken` (같은 목록에 있음 — **둘 다 필요**)
4. 터미널에서 실행:
   ```
   python tools/create_instagram_session.py
   ```
   안내에 따라 두 값을 붙여넣는다 (입력값은 화면에 표시되지 않는다).
5. `[완료] 세션 파일 저장:` 메시지를 확인하고 **봇을 재시작한다.**

## 언제 필요한가

- 비공개 계정 게시물처럼 embed로 못 가져오는 링크를 지원해야 할 때.
- 공개 게시물인데도 계속 **"이 게시물은 삭제됐거나 접근할 수 없습니다"**가 나올 때
  (Instagram이 embed 경로를 막았을 가능성 — 이 경우 세션 폴백이 구원투수가 된다).
- Instagram 비밀번호를 바꾸면 기존 세션이 무효화되므로 다시 만들어야 한다.
- `saved/instagram_errors.log`에 `세션 인증 거부`가 반복되면 만료된 것이다.

## 알아둘 것

- 파일이 만들어졌다고 세션이 유효한 건 아니다. 잘못된 쿠키도 저장까지는 된다.
  갱신 후에는 테스트방에 공개 게시물 링크 하나를 올려 이미지가 오는지 확인한다.
- 봇이 켜진 상태에서 세션 파일만 바꾸면 최대 10분 내 자동 재로드된다.
  즉시 반영하려면 봇을 재시작한다.
- 세션 파일과 `config/api_key.json`은 gitignore 대상이다. 커밋되지 않는지
  푸시 전에 `git status`로 확인한다.
- "요청 제한" 안내가 나오면 갱신하지 말고 잠시 기다린다 — 반복 시도는
  차단을 악화시킨다.
